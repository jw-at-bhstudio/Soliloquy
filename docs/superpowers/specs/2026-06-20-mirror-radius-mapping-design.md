# 镜像半径映射设计

## 背景

镜像模块合并后，当前同心圆半径主要由“当前显示序列位置 + 能量偏移”决定，而不是由频率或谐波槽位决定。这带来两个核心问题：

1. 取消某一轨后，其余轨会补位，导致 `1,2,3` 与 `1,3,4` 的视觉差异被压缩。
2. “自适应能量轨道排布”只是在当前序号式半径基础上做偏移，无法形成从“纯频率”到“纯能量排序”的清晰连续轴。

本设计用于恢复镜像模块的可解释性、可区分性和可逆性。

## 目标

### 目标 1：取消显隐不改变其他轨位置

所有轨道的位置必须先被完整计算，再决定是否绘制。勾选状态只影响显示，不再影响其余轨道的圈层位置。

### 目标 2：提供两种半径模式

镜像模块新增两种可切换的半径模式：

1. `绝对频率`
2. `相对谐波`

两种模式都必须满足“取消某轨后其他轨不补位”。

### 目标 3：重定义能量轴

现有“自适应能量轨道排布”滑杆改为一个真正的连续插值轴：

- `0%`：完全按频率或谐波位置
- `100%`：完全按能量决定内外圈
- `0% ~ 100%`：在两端之间平滑滑动

## 非目标

- 本次不引入第三种“混合频率 + 谐波”的半径模式。
- 本次不引入更复杂的频率压缩曲线，如对数映射或高频软压缩。
- 本次不引入更多高级统计项供用户切换，先以一套简单可解释规则落地。

## 半径模式

### 模式 A：绝对频率

#### 定义

每条轨道先计算一个代表频率 `f_track`，再使用全局固定频率尺把该频率映射到 `radiusMin ~ radiusMax`。

#### 默认固定频率尺

- `frequencyMin = 80Hz`
- `frequencyMax = 2000Hz`

默认规则：

- `80Hz -> radiusMin`
- `2000Hz -> radiusMax`

这两个值必须在界面中可编辑。

#### 超出范围处理

第一版采用最简单规则：`clamp`

- 低于 `frequencyMin` 的频率，按 `frequencyMin` 处理
- 高于 `frequencyMax` 的频率，按 `frequencyMax` 处理

原因：

- 可解释
- 可逆
- 便于先验证整体结构

如果后续发现大量高倍频挤在外圈，再单独讨论引入压缩映射。

#### 代表频率

第一版使用该轨有效帧频率的平均值作为 `f_track`。

对镜像模块当前数据结构而言，这等价于：

- 第 1 轨使用平均 `F0`
- 第 2 轨使用平均 `2 * F0`
- 第 3 轨使用平均 `3 * F0`
- 以此类推

### 模式 B：相对谐波

#### 定义

每条轨道使用固定谐波槽位，而不是按实际频率映射。

例如：

- `F0` 固定在第 1 圈
- `2x` 固定在第 2 圈
- `3x` 固定在第 3 圈

无论是否勾选显示，该槽位都不改变。

#### 作用

该模式用于获得结构稳定、便于对照的版本。相比绝对频率模式，它会减少“不同人基频不同”带来的整体圈层差异。

## 能量轴重定义

### 两端定义

先独立计算两套半径：

1. `r_freq`
2. `r_energy`

然后通过滑杆值 `t` 做线性插值：

`r = (1 - t) * r_freq + t * r_energy`

其中：

- `t = 0` 对应 `0%`
- `t = 1` 对应 `100%`

### 频率端半径 `r_freq`

#### 在绝对频率模式下

`r_freq` 来源于固定频率尺映射。

#### 在相对谐波模式下

`r_freq` 来源于固定谐波槽位映射。

### 能量端半径 `r_energy`

第一版不直接把能量值映射为连续半径，而是先做排序，再把排序结果分配到固定槽位。

规则：

1. 计算每条轨道的平均能量
2. 从低到高排序
3. 将排序结果映射到从内到外的固定槽位

于是：

- 能量最低的轨更靠内
- 能量最高的轨更靠外

这样在 `100%` 端时，可以真正实现“谁能量高谁在外面”。

### 为什么使用排序而不是直接能量值映射

如果直接把能量值映射为连续半径，会出现：

- 样本之间整体尺度不稳定
- 外圈或内圈利用不充分
- 难以明确达到“纯能量决定圈层”的端点效果

用排序分配槽位更稳定，也更接近你描述的极端端点。

## 显示与位置解耦

### 旧问题

当前系统是“先决定显示哪些轨，再按当前显示序列重新分配圈层位置”。

这导致：

- 去掉 2 轨后，3 轨会回到原先 2 轨的位置
- 图形区分度显著下降

### 新规则

新系统改为：

1. 所有可用轨先计算完整位置
2. 每轨拿到稳定的 `r_freq`、`r_energy` 和最终 `r`
3. 勾选状态仅决定是否绘制

因此：

- `1,2,3` 与 `1,3,4` 的位置关系不再塌缩
- 取消某轨不会引发其他轨补位

## UI 变更

### 新增控件

镜像参数区新增：

1. `半径模式`
   - `绝对频率`
   - `相对谐波`

2. `频率下限`
   - 默认 `80`

3. `频率上限`
   - 默认 `2000`

### 保留并改造的控件

现有 `自适应能量轨道排布` 滑杆保留，但语义更新为：

- `0%`：完全频率/谐波
- `100%`：完全能量排序

其说明文案也要同步更新，避免用户误解为只是“谁强谁略微向外扩散”。

### 保持不变的控件

- 左右侧精确选轨
- 左右侧 `分轨 / 加合`
- `radiusMin / radiusMax`
- `amplitudeScale`
- `waveDensityMultiplier`

## 绘图尺寸与边界

### 原则

这次改动必须继续使用相对尺寸体系，确保 8 倍频及更多轨道在默认参数下不会轻易超出绘图区。

### 第一版做法

- 半径仍然只在 `radiusMin ~ radiusMax` 内分配
- 不因高频而自动突破 `radiusMax`
- 高频轨在绝对频率模式下如果超出 `frequencyMax`，先钳制到外圈边界

这意味着：

- 画布边界可控
- 高频拥挤问题暂时接受
- 如果后续观察到外圈过密，再单独做高频压缩策略

## 数据流与实现边界

### 新增配置项

`RenderConfig` 新增：

- `radiusMode`
- `frequencyMin`
- `frequencyMax`

### 新增计算步骤

在镜像绘图坐标计算前，先生成完整的轨道槽位信息：

1. 轨道代表频率
2. 轨道平均能量
3. `r_freq`
4. `r_energy`
5. 插值后的最终 `r`

然后再进入左右半边的实际点位绘制。

### 与选轨逻辑的关系

选轨逻辑只消费已经计算好的轨道位置，不再参与位置分配本身。

## 错误处理与边界情况

### 边界情况 1：频率上下限非法

如果用户输入：

- 下限小于等于 0
- 上限小于等于下限

则界面应自动修正为合法范围，避免出现除零或倒序映射。

### 边界情况 2：频率全部越界

如果某些轨道全部高于 `frequencyMax` 或低于 `frequencyMin`，第一版允许它们被压在边界圈层。

### 边界情况 3：能量相同

若多条轨能量极其接近或完全相同，排序时使用稳定排序；在相同能量下保留原轨道顺序，避免随机跳动。

## 测试

需要新增或更新测试以覆盖以下行为：

1. 取消某轨后其他轨位置不改变
2. 绝对频率模式下，固定频率尺决定固定半径
3. 相对谐波模式下，轨道槽位稳定
4. 能量滑杆在 `0%` 与 `100%` 端点行为正确
5. 频率上下限非法时可被修正
6. 同一批轨在不同勾选组合下，位置数组保持不变

## 成功标准

当以下条件同时满足时，认为本次设计达标：

1. `1,2,3` 与 `1,3,4` 在视觉上能明显区分
2. 取消任意轨后，其余轨位置不补位
3. `绝对频率` 模式能体现不同样本基频差异
4. `相对谐波` 模式能提供稳定对照
5. 能量滑杆的两端定义明确且可感知
6. 默认参数下画面不会轻易超出绘图区
