# Soliloquy Visual Structure Unification Design

## Background

品牌、字体、字号与交互文案已经统一到 `Soliloquy / 独白` 体系，但页面结构与视觉层级仍存在明显不一致：

- `Analyzer` 页头过重，承担了过多统计与视觉强调
- `Mirror` 页头较轻，但与 `Analyzer` 还没有共享同一套页面骨架
- 两个页面的内容容器、边距、主区网格、面板语义、按钮层级、边框强度和图标使用规则尚未收敛

当前最主要的问题不是功能缺失，而是页面中“谁应该最醒目”还不够稳定，导致视觉噪声偏高，产品一致性不足。

## Goal

- 统一 `Analyzer` 与 `Mirror` 的页面骨架
- 建立明确的视觉层级：`可视化区域 > 主按钮 > 页头 > 其他信息`
- 将 `Analyzer` 页头降重，并让两个页面都采用轻页头
- 统一内容容器、页面边距、主区间距与网格语法
- 统一面板语义、边框强度、底色层级与按钮层级
- 减少图标滥用，移除装饰性 icon

## Non-Goals

- 不修改核心业务逻辑
- 不新增页面或流程
- 不重新定义品牌、字体、字号与术语规则
- 不改变主要工作流顺序
- 不追求“展览感”或“仪表盘感”的极端风格

## Core Decision

本轮视觉统一采用 `中间态`，但以 `一致优先` 为最高原则。

这意味着：

- 不追求花哨的视觉强化
- 不保留控制台式的高噪声结构
- 保留工具效率，但让页面更安静、更统一、更有秩序

## Visual Hierarchy

全站页面视觉层级固定为：

1. 可视化区域
2. 主按钮
3. 页头
4. 其他信息

### Hierarchy Rules

- 可视化区域必须是页面中最强的视觉中心
- 主按钮必须是除可视化区域外最醒目的元素
- 页头只负责定位当前页面与一句说明，不承担强调任务
- 统计、摘要、说明、次级参数和辅助信息全部后退

## Page Shell

### Shared Container

`Analyzer` 与 `Mirror` 必须共用同一套页面容器规则：

- 同一最大内容宽度
- 同一水平边距
- 同一顶部起始线
- 同一页头与主区对齐方式

### Shared Vertical Rhythm

两个页面必须共用同一套纵向节奏：

- 页头到状态条的间距一致
- 状态条到主区的间距一致
- 面板之间的常规间距一致
- 大区块之间仅允许 1 档更大的间距

不得再出现 `Analyzer` 与 `Mirror` 拥有各自独立边距体系的情况。

## Header Design

### Header Role

页头只承担两项职责：

- 当前页面名称
- 当前页面的一句用途说明

### Header Rules

- 页头必须轻
- 页头不能卡片化
- 页头不能使用大图标块
- 页头不能承载统计信息
- 页头不能成为页面最强视觉区域

### Header Content

页头只允许包含：

- 一级页标题
- 一句二级说明
- 极少量页级动作

### Header Consistency

`Analyzer` 与 `Mirror` 的页头必须采用同一骨架：

- 左侧：标题与说明
- 右侧：少量页级动作
- 下方：如需状态信息，进入独立状态条

### Specific Decision

- `Mirror` 当前“轻页头”的方向是正确基线
- `Analyzer` 必须向 `Mirror` 收敛，而不是反向强化 `Mirror`

## Status Strip

### Role

状态条位于页头之后、主区之前，只承担快速告知作用。

### Rules

- 状态条视觉强度低于主按钮
- 状态条视觉强度不高于页头
- 状态条内部标签极简
- 状态条数值统一使用等宽字体
- 状态条结构和样式在两个页面中保持一致

### Analyzer Status

`Analyzer` 状态条承载：

- `总数`
- `排队`
- `处理中`
- `可导出`

### Mirror Status

`Mirror` 状态条承载：

- `来源`
- `时长`
- `帧数`
- 必要时的 `模式`

## Main Workspace

### Shared Rule

两个页面都必须让主工作区显著强于页头与辅助信息。

### Analyzer Workspace

`Analyzer` 保留三栏布局，但必须重新明确主次关系：

- 左栏：输入与参数
- 中栏：当前结果与波形
- 右栏：轨道与摘要

中栏必须是 `Analyzer` 的视觉中心。

### Mirror Workspace

`Mirror` 保留“主可视化区 + 右侧控制区”的布局，但必须纳入同一页面语法：

- 左侧画布区是主面板
- 右侧控制区是次级面板组
- 控制区不能呈现为一个独立的外挂系统

## Panel System

### Panel Principle

面板只服务于结构，不负责抢视觉中心。

### Panel Types

只保留两类面板：

- `工作面板`
- `轻信息面板`

### Panel Rules

- 相同圆角
- 相同边框逻辑
- 相同 padding 节奏
- 相同标题位置
- 相同说明文字层级

### Panel Title Rules

- 面板标题默认无图标
- 面板标题不强化
- 面板标题只使用短中文名词
- 面板标题的视觉强度低于主按钮与页头标题

### Page Mapping

`Analyzer` 工作面板：

- `输入`
- `参数`
- `波形`
- `轨道`
- `摘要`

`Analyzer` 轻信息面板：

- `原理`

`Mirror` 工作面板：

- `画布`
- `数据`
- `轨道`
- `形变`
- `绘制`
- `导出`

`Mirror` 轻信息面板：

- `说明`

## Button System

### Role

主按钮是除可视化区域外最醒目的操作元素。

### Levels

只保留三级按钮：

- `主按钮`
- `次按钮`
- `弱按钮`

### Primary Button

- 使用实体反白
- 保持最高对比
- 数量必须受控
- 默认不使用图标

### Secondary Button

- 透明底 + 细边框
- 与主按钮同高
- 视觉强度明显低于主按钮

### Weak Button

- 更轻的边框或文本化
- 只用于辅助动作
- 不得与主按钮竞争视觉中心

### Action Priority

应被定义为主按钮的动作包括但不限于：

- `导入文件`
- `开始试听`
- `导出 JSON`
- `复制 SVG`

页级动作如 `返回分析`、`说明` 默认不应成为主按钮。

## Border And Surface System

### Border Rules

边框只允许两档语义：

- `常规边框`
- `强调边框`

### Border Restrictions

- 默认全部使用常规边框
- 只有激活态、选中态或极少数关键容器可使用强调边框
- 不得混用不同色相的边框体系

### Surface Rules

表面深度只允许三层：

- `页面底`
- `面板底`
- `强调面板底`

### Surface Restrictions

- 不允许页面中散落多组不同黑值
- 不允许不同模块各自维护底色规则
- 强调必须靠结构与层级，不靠更花的底色

## Icon Policy

### Principle

图标必须去装饰化。

### Rules

- 导航默认不使用图标
- 页头默认不使用图标
- 面板标题默认不使用图标
- 主按钮默认不使用图标
- 只有 `返回`、`关闭`、`播放/暂停` 等高识别动作可保留极少量图标

### Consistency

- 图标尺寸统一
- 图标颜色跟随文字
- 图标不允许单独成为视觉重点

## Page-Specific Adjustments

### Analyzer

- 将当前过重页头拆为轻页头
- 将统计胶囊下沉到状态条
- 将视觉中心重新交还给中区结果与波形区域
- 清理页头中的重图标与过强强调

### Mirror

- 保持轻页头方向不变
- 不增强页头存在感
- 让 `开始试听`、`复制 SVG`、`导入 JSON` 等主动作在页头与控制面板中拥有更高优先级
- 将控制区整理为统一的次级面板组，而非独立外挂控制台

## Acceptance Criteria

完成后应满足以下结果：

- 两个页面看起来属于同一个产品
- `Analyzer` 不再拥有过重页头
- `Mirror` 不再显得像另一个独立系统
- 主按钮优先级明显高于页头与辅助信息
- 非可视化信息整体后退
- 页面网格、边距、面板、边框、按钮和图标规则均可在两个页面中一眼看出一致性

## Verification

后续实现验收必须至少检查：

- 两页页头是否都为轻页头
- 两页主内容容器是否一致
- 状态条是否拥有统一语义与样式
- 主按钮是否成为非可视化区域中的第一视觉优先级
- 面板标题是否默认无图标
- 多余装饰性 icon 是否已清理
- 边框和底色是否已收敛到统一语义

## Implementation Boundary

本轮后续实现只涉及视觉结构与样式统一：

- 页面容器
- 页头
- 状态条
- 主区网格
- 面板系统
- 按钮层级
- icon 使用规则

不重新修改已完成的品牌、字体、字号和术语基础规则。
